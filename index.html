Solar — how it works

Smart matching (the point of the tool)

Entities arrive two ways: Paste Text (free-text intelligence passages) and Import CSV. Phones are normalised to E.164 (07… ⇄ +447…), emails canonicalised (gmail dot/plus rules), names fuzzy-matched with a nickname table (Geoff ⇄ Geoffrey). Matches are suggested, never auto-merged — you decide on the review screen or in Deconflict.

Review screen

Left: your text with colour-coded extractions. Right: one card per entity — edit labels, choose match-vs-new, opt in to inferred relationships (⇄ swaps direction). Confidence is tiered HIGH / MED / LOW, never a fake percentage. Ambiguous DD/MM dates are flagged, not guessed.

Views

Chart (vis-network: drag, double-click to focus), Map (gazetteer-located entities; addresses pin to their locality), Timeline (dated events; click to select). All three select together.

Provenance

Every entity carries an NCA 3×5×2 grading (A–C / 1–5 / P–C), source reference and a full audit trail — see the inspector.

Data

Everything stays in this browser (localStorage + JSON save/load). The basemap tile request is the only network call. Exports: case JSON, chart PNG with protective marking, i2 ANX XML.
